SEKA PASS プライバシーポリシー

最終更新日: 2026年7月18日

基本方針

SEKA PASS（以下「本アプリ」）は、ユーザーのプライバシーを尊重して設計されています。訪問国の記録などの主要なデータは端末内にのみ保存されます。ただし「友達」機能をご利用になる場合、および広告表示のために、一部のデータが外部サービスへ送信されます。詳細は以下をご確認ください。

端末内にのみ保存されるデータ

以下の情報は、お使いの端末内にのみ保存され、外部に送信されることはありません。

  • 訪問国の記録（国名・入出国日・移動手段・メモ）
  • 行きたい国のリスト
  • プロフィール情報（氏名・旅券番号・生年月日・発行日・有効期限・顔写真など）

友達機能について（任意機能）

「友達」タブの機能を利用しない限り、以下のデータは送信されません。

友達機能では、匿名の認証情報を用いてGoogle Firebase（Google LLCが提供するサービス）に以下の情報を保存し、あなたが追加した友達との間で共有します。

  • 友達専用のニックネーム（旅券情報の氏名とは別に、任意で設定するもの）
  • 友達コード（アプリが自動生成するランダムな識別コード）
  • 訪問した国のコード・訪問年（国名・年のみで、メモや移動手段などの詳細は送信されません）
  • 友達関係の情報（誰と友達になっているか）

これらの情報は匿名アカウントに紐づけて保存され、氏名・連絡先などの個人を特定できる情報とは結び付けていません。友達機能を使わない場合、これらの送信は発生しません。

QRコード読み取り（カメラ）

友達コードをQRコードで交換する際にのみ、カメラへのアクセスを求めます。読み取った映像は保存・送信されず、QRコードの内容（友達コード）の読み取りのみに使用されます。

広告について

本アプリはGoogle AdMob（Google LLCが提供する広告サービス）を利用して、バナー広告および全画面広告を表示します。広告配信のため、Google AdMobが広告識別子（IDFA）や使用状況データを収集する場合があります。収集される情報の詳細は、Googleのパートナー向けポリシーをご確認ください。iOSでは、初回起動時にトラッキングの許可を求めるダイアログが表示されることがあります。許可しない場合でも、パーソナライズされていない広告が表示されます。

外部との通信について

本アプリは以下の場合にもインターネット通信を行います。

  • 国旗画像の取得: 国旗の表示に flagcdn.com の画像を使用します。このとき送信されるのは国コード（例: JP）のみで、個人情報は含まれません。
  • 国の風景写真の取得: 情報画面のカルーセル表示のため、Wikipedia（Wikimedia）の画像APIから国名に基づく画像を取得します。
  • Wikipediaへのリンク: 国の詳細情報を見るためにWikipediaを開く場合、標準ブラウザで該当ページを表示します。
  • Trip.comへのリンク: 航空券検索のためにTrip.comを開く場合、標準ブラウザで該当ページを表示します。これはアフィリエイトリンクとして機能する場合がありますが、本アプリからTrip.comへユーザーの個人情報が送信されることはありません。

Face ID / Touch ID

パスポート情報の保護のために生体認証を利用できます。認証処理はすべてOS（iOS）が行い、本アプリが生体情報にアクセスすることはありません。

写真ライブラリ

プロフィールの顔写真を設定する場合のみ、写真ライブラリへのアクセスを求めます。選択した写真は端末内にのみ保存されます。

データの削除

アプリを削除すると、端末内に保存されたデータはすべて削除されます。友達機能で外部（Firebase）に送信したデータの削除をご希望の場合は、下記のお問い合わせ先までご連絡ください。

お問い合わせ

本ポリシーに関するお問い合わせは、App Storeのサポート欄に記載の連絡先までお願いします。

変更について

本ポリシーを変更する場合は、アプリのアップデートおよび本ページで通知します。
